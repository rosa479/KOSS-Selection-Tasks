# Dissecting a Chess Engine

## Background

Chess is a game with infinite-ish possibilities (10^120 to be exact), yet modern engines like Stockfish and Lc0 find the perfect moves in a snap. It feels like black magic. 

You must undertake the journey to investigate the mysterious workings of these engines, and possibly create one yourself!
## Task Description

- Research how computers "think" about chess, specifically the following :
  - **Board Evaluation** - how a computer scores a position
  - **MiniMax Algorithm** - how the engine looks ahead in time
  - **Alpha-Beta Pruning** - how the engine reduces the possibilities to be calculated
  - and more if you wish to :D

- For brownie points, write a basic and functional chess engine. You may use a library for UI and legal move generation (or maybe not for double brownie points). Your focus should be implementing a custom evaluation function and a Minimax search function (depth 2 or 3) that allows the computer to play a valid game against a human in the terminal.

- A GitHub repository which contains your code (if any) along with a .md file or any other form of documentation containing your learning process is the expected end product.


## Tech-Stack
You can use any programming language and tools that are available; however, make sure you are ready to explain why you chose them and the pros/cons that you encountered while using them.
## Learning from the Task
While you must have some programming experience to do this task, we know beforehand that it will be difficult for people who don't have much experience with search algorithms or game theory. So, you are encouraged to google the technical stuff that you encounter here, and use technical blogs to understand and practically implement it. Don't get lost in the theoretical sea—there are infinite topics in AI and heuristics which are impossible to know completely. You should judiciously use your time to maintain a balance of theoretical understanding and practical implementation. We are more curious about your thinking ability and practical utility than the final result.
